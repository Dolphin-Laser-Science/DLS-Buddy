# Synthetic Stress corpus — expected outcomes

Adversarial test files that probe robustness gaps in the parser, data-model,
and analysis layers. Generated by a seeded, reproducible synthetic
stress-data generator and driven by the regression test suite, which
asserts each file is handled as its **Outcome** says.

## Outcome legend

- **REJECT** — Already fails loud with a clear error at parse/build. Green today; guards against a future refactor silently accepting it.
- **OK** — A quirky-but-valid file is already accepted correctly. Green today; guards against a refactor breaking that handling.
- **FLAG** — Analyzes, but the correct diagnostic already fires (unreliable / apparent / non-physical). Green today; guards the flag.
- **KNOWN-DEFECT** — A real current gap: crashes, hangs, or silently mis-analyzes. Carried as an xfail that asserts the DESIRED hardened behavior. Deterministic gaps use strict=True, so the suite turns red the moment the gap is fixed -- prompting promotion to REJECT/FLAG; the two fit-outcome-dependent cumulant cases stay non-strict.
